Project 1 // BMAD Learning Vehicle

Swiss Clock
Homage Screensaver

A macOS screensaver with multiple analog clock face designs that pay homage to famous Swiss watches. Real shippable open-source project, used as the on-ramp to BMAD before tackling the trading system.

PROJECT Open source · MIT TARGET macOS .saver bundle EFFORT ~2 weeks evenings
Why this project
It exercises every BMAD agent on something real. Analyst frames the audience and constraints. PM scopes the dial set and packaging. Architect designs the plugin system. SM/Dev/QA ship each dial. The plugin pattern you build here is the same pattern you'll reuse for trading strategies. And at the end you have a shippable artifact that real horology enthusiasts might actually use.
01 // Legal scope

Homages, Not Copies

The single most important product decision. Get this right and you have something you can publish freely. Get it wrong and you can get cease-and-desisted by some of the most well-funded legal teams in Switzerland.

What's allowed
Inspired-by silhouettes and design vocabulary. Watch case shapes, bezel motifs (octagons, fluted, tachymeters), dial textures (tapisserie patterns, sunburst, snail), hand styles (sword, dauphine, mercedes), color palettes, indices styles, complications layouts. These are design language, not protected expression. A "homage" community has existed in horology for decades — done with restraint and credit, it's a celebration of design history.
What's NOT allowed
Trademarked names, logos, signed dials, distinctive trademarked elements. No "Rolex" anywhere. No five-pointed crown. No Patek Calatrava cross. No Audemars Piguet "AP" monogram. No Tudor shield. No signed maker's name on the dial. No model names — no "Submariner", "Speedmaster", "Nautilus", "Royal Oak" — these are trademarks. Refer to dials in your code and UI by your own names that evoke the inspiration without copying the trademark.
Documentation
In your README, credit each homage's inspiration explicitly: "Geneva Octagon is a homage to the integrated-bracelet luxury sports watch genre pioneered by the 1972 Audemars Piguet Royal Oak." Acknowledgment + transformation = legitimate homage. Pretending it's original = problem.
02 // v1 dial set

Five Initial Designs

Each is a homage to a recognizable genre or piece. Names are deliberately neutral so no trademarks are infringed. The variety is intentional — covers different eras, styles, and aesthetics, exercises the plugin system, and gives users something to pick between.

Bahnhof
Mondaine / SBB
The minimalist Swiss railway clock — the original reference from davenicoll's project. Stark white face, black indices, red second hand with the iconic stop-and-pause "loving" tick at 12.
  • Hands: bar-style black, red "lollipop" seconds
  • Indices: bold black bars, no numerals
  • Motion: stop-2-second pause at 12
  • Palette: white / black / Swiss red
Geneva Octagon
Royal Oak genre
Integrated-bracelet luxury sports homage. Tapisserie-textured dial, octagonal bezel hint at the edge of the canvas, applied baton indices.
  • Hands: tapered baton, lume-line down center
  • Indices: applied batons, raised effect
  • Texture: tapisserie waffle pattern
  • Palette: blue / steel / off-white
Lunaris
Speedmaster genre
Tool-watch chronograph aesthetic. Black dial, tachymeter bezel ring, three sub-dials at 3/6/9. The sub-dials are decorative in screensaver mode but become active functions in display modes that show seconds or date.
  • Hands: straight stick, white with lume
  • Indices: applied baton + Arabic 12
  • Sub-dials: three, white-on-black
  • Palette: black / white / orange accent
Calatrava
Dress watch genre
Pure dress watch — the most restrained option. Silver opaline dial, gold dauphine hands, leaf-shape second hand, applied Roman numerals at 12/3/6/9. The kind of watch that whispers.
  • Hands: dauphine, gold/yellow
  • Indices: Roman + applied batons
  • Texture: sunburst opaline
  • Palette: silver / gold / black hands shadow
Submersible
Dive watch genre
Tool-watch dive aesthetic. Unidirectional bezel ring around the edge with elapsed-time markings, oversized lumed indices, mercedes-style hour hand. The bezel rotates once per hour as a clever ambient timer.
  • Hands: mercedes-style hour, broad-arrow minute
  • Indices: oversized circles/triangles, lume effect
  • Bezel: animated unidirectional, 0–60 markings
  • Palette: matte black / lume green / steel
03 // Tech decision

Native Swift or WebKit?

A real architectural decision the architect agent will walk you through. Both work. They have different tradeoffs.

Native Swift / SwiftUI

Recommended for v1

Subclass ScreenSaverView, render each dial in Core Graphics or SwiftUI Canvas. Each dial is a Swift type implementing a DialRenderer protocol.

Pros

  • Native perf, no WebView overhead
  • Same idioms as macOS itself
  • App Store distribution path if you want it
  • Sharper text rendering, retina-correct

Cons

  • Swift learning curve if you're new to it
  • Core Graphics drawing is verbose
  • Xcode is mandatory

WebKit-hosted HTML/SVG

Faster start

ScreenSaverView hosts a WKWebView. Each dial is HTML/CSS/SVG/JS. Reference project (davenicoll's) uses this approach. Faster to prototype, especially if you're more comfortable with web tech.

Pros

  • SVG is forgiving for clock graphics
  • CSS animations for hand sweeps
  • Reload to iterate without rebuild
  • You probably know it already

Cons

  • WebView memory overhead
  • Animation framerate less predictable
  • Anti-aliasing quirks at retina densities
Recommendation
Start with WebKit-hosted SVG if you want to ship fastest. Move to native Swift for v2 if performance bothers you or you want App Store distribution. The plugin system you'll design should let you swap rendering tech per-dial — useful escape hatch.
04 // BMAD stories

Story Outline

Roughly how the SM agent will shard the work. Each story is one Claude Code session. Use this as a starting point, let SM refine.

S1
Project skeleton + .saver bundle. Xcode project, ScreenSaverView subclass, hello-world dial that renders a circle and ticking hand. Installable as .saver.
S2
Dial plugin protocol. DialRenderer protocol defining the contract: dimensions, dirty regions, draw at time t, preferences schema. First concrete dial conforms to it.
S3
Time + animation pipeline. NTP-synced wall clock (no drift over the screensaver session), hand position interpolation, smooth vs ticking second-hand modes.
S4
Bahnhof dial. Including the famous stop-2-second pause at 12 — actually a subtle animation detail that's harder than it looks.
S5
Geneva Octagon dial. Tapisserie texture as an SVG pattern, octagonal hint at the canvas edge, applied indices with raised shadow effect.
S6
Lunaris dial. Three sub-dials + tachymeter ring. Sub-dials are visual-only in v1.
Optional v2: live running-seconds sub-dial.
S7
Calatrava dial. Roman numerals + sunburst pattern. Tests your design discipline — the simplest dial visually but the hardest to nail.
S8
Submersible dial. Animated rotating bezel ring + mercedes hands. Lume effect for indices.
S9
Preferences pane. macOS standard preferences UI for the screensaver. Dial selector with previews. Optional: per-dial preferences (e.g., bezel rotation speed).
S10
Multi-monitor + retina. Render correctly on multiple displays at different DPIs. Edge case that bites if not handled explicitly.
S11
Packaging + README. Build script that produces a signed .saver bundle. README with screenshots, installation, design notes per dial.
S12
GitHub release. Tag v1.0, attach .saver to the release. Optional: post to MacRumors forums, HN, r/Watches.
05 // What you'll learn

BMAD Muscles This Builds

Every lesson here transfers directly to the trading system.

Plugin systems
The DialRenderer protocol is structurally identical to the Strategy base class for trading. Each dial is independent. Adding a new dial is dropping a folder. Same pattern, lower stakes.
Product scope discipline
"Five dials in v1" is the same skill as "two strategies in v1" for the trading system. The PM agent will push back if you scope too wide. Learning to push back well is exactly what saves you on the trading system.
Real-world constraints
The legal homage-vs-copy line is real friction that BMAD's planning agents handle well if you bring it up. Same shape as the trading system's surveillance/compliance constraints — different domain, identical skill.
SM/Dev/QA rhythm
By the time you finish 12 stories on the screensaver, the rhythm is automatic. You're reading every story, every diff. You know what a good story looks like and what a too-big story looks like. That fluency is the entire point.
